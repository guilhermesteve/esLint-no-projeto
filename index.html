Configurando ESLint em um projeto Node.js com VS Code

Este tutorial irá guiá-lo na configuração do ESLint em um projeto Node.js utilizando o VS Code como editor de código.

Pré-requisitos

Para seguir este tutorial, você precisa ter o seguinte instalado no seu computador:

  • Node.js (versão 12 ou superior)
  • VS Code (ou qualquer outro editor de sua preferência)

Passo 1: Inicializando o projeto Node.js

Antes de começar, certifique-se de ter o Node.js instalado em sua máquina. Em seguida, abra o terminal e execute o seguinte comando:

mkdir [meu-projeto]
cd [meu-projeto]
npm init -y

Passo 2: Instalando o ESLint

Para instalar o ESLint, execute o seguinte comando no terminal:

npm i -D eslint

Passo 3: Configurando o ESLint

Crie um arquivo chamado .eslintrc.json na raiz do projeto com o seguinte conteúdo:

{
  "extends": "@rocketseat/eslint-config/node"
}

Isso irá configurar o ESLint com as regras recomendadas para projetos Node.js.

Passo 4: Instale uma configuração compartilhada do ESLint

O ESLint tem muitas configurações disponíveis para escolher, mas é recomendável usar uma configuração compartilhada para garantir que seu projeto esteja seguindo as melhores práticas de desenvolvimento.

A Rocketseat, uma empresa brasileira de educação em tecnologia, criou algumas configurações compartilhadas para o ESLint que são amplamente utilizadas na comunidade brasileira de desenvolvedores. Você pode instalá-las executando o seguinte comando no terminal integrado do VS Code: 

{
        npm i -D eslint @rocketseat/eslint-config
      }

Passo 6: Instale a extensão ESLint no VS Code

Você pode adicionar utilizando o seguinte comando: 

{
        code --install-extension dbaeumer.vscode-eslint
      }

Passo 7: Configuração VsCode

configurando o ESLint no arquivo de configuração do VS Code:` > settings.json `

{
        {
            "editor.codeActionsOnSave": {
              "source.fixAll.eslint": true
            }
          }
    }

Passo 7: Configuração package.json

Crie um script para ser executado quando tiver que ajustar em lote os arquivos da aplicação

{
        {
            "scripts": {
              "lint": "eslint . --ext .ts,.tsx"
            }
          }
    }
